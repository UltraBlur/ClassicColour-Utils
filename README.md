# PR788 光谱采集工具使用说明

本工具用于通过串口连接 `PR788`，采集光谱数据，保存为 `CSV`，并在界面中查看当前测量结果、`SPD` 预览图和颜色预览。

当前图形界面位于 `ENV_PR788` 目录下，推荐通过 `ENV_PR788/main.py` 启动。

## 1. 使用前准备

请先确认以下事项：

- `PR788` 已正确连接到电脑
- 已确认设备对应的串口号可以被系统识别
- Python 环境中已安装本工具所需依赖

建议至少具备这些依赖：

- `pyserial`
- `PyQt5`
- `colour`
- `numpy`

如果你之前装过错误的 `serial` 包，而不是 `pyserial`，串口功能会出问题。  
可以使用下面的方式修正：

```powershell
pip uninstall serial
pip install pyserial
```

## 2. 启动方式

推荐直接从 `ENV_PR788` 根目录启动：

```powershell
python ENV_PR788\main.py
```

如果使用指定 Conda 环境，例如：

```powershell
C:/Users/gaohu/.conda/envs/EleColor-Spec/python.exe c:/Users/gaohu/Desktop/ClassicColour-Utils/ENV_PR788/main.py
```

## 3. 界面功能说明

界面分为左右两部分。

### 左侧：控制区

左侧主要负责设备连接、保存设置和测量控制。

包括：

- 串口选择
- `PR788` 连接 / 断开
- 输出路径设置
- 文件名模板设置
- 自动编号设置
- 测量按钮

### 右侧：结果区

右侧主要负责显示当前一次测量的结果。

包括：

- 当前文件名和保存路径
- `sRGB` 颜色预览块
- `SPD` 预览图
- 色度与白点参数
- 日志区

## 4. 典型使用流程

### 第一步：启动程序

运行 `ENV_PR788/main.py` 后，界面会启动。

### 第二步：选择串口

在左上角串口下拉框中选择当前 `PR788` 对应的串口。  
如果刚插上设备，可以点击 `Refresh` 重新扫描串口。

### 第三步：连接 PR788

点击 `Connect PR788`。

点击后，软件会做的是：

- 打开当前串口
- 向 `PR788` 发送远程模式相关指令
- 在日志区记录当前操作

重要说明：

- 软件界面中的状态只表示“串口已打开且 remote 指令已发送”
- 这并不等于已经百分之百确认设备真正进入远程模式
- 是否真正连接成功，请以 `PR788` 设备屏幕上的显示为准
- 只有当 `PR788` 自身界面明确进入 `REMOTE MODE` 时，才应认为远程连接成功

如果连接失败：

- 请检查串口是否被其他软件占用
- 请检查串口号是否选错
- 请确认设备是否已经正常供电

### 第四步：设置输出目录

在 `Output Folder` 中选择保存 `CSV` 文件的目录。

### 第五步：设置文件名模板

文件名模板目前支持两个占位符：

- `{counter}`
- `{timestamp}`

例如：

```text
{counter}_{timestamp}
```

或者：

```text
sample_{timestamp}
```

界面会自动预览最终的文件名和完整保存路径。

### 第六步：设置自动编号

如果希望文件名自动带编号，可以勾选：

```text
Enable auto counter
```

可设置：

- `Start`
  起始编号
- `Step`
  每次测量后编号递增的步长
- `Current`
  当前将要使用的编号

点击 `Reset To Start` 可以把当前编号恢复到起始值。

说明：

- 当前自动编号完全由界面内部管理
- 不再依赖外部 `txt` 文件

### 第七步：开始测量

点击：

```text
Capture Measurement
```

程序会：

1. 触发 `PR788` 测量
2. 读取返回数据
3. 保存为 `CSV`
4. 更新右侧结果

开始测量前，请先确认：

- `PR788` 设备屏幕已经进入 `REMOTE MODE`

## 5. 关于等待时间

当前测量读取默认不会主动超时。

也就是说：

- 如果 `PR788` 某次读数很慢
- 程序会继续等待
- 不会因为固定的 30 秒或 60 秒限制而自动报错

这适合那些单次测量时间较长的场景。

## 6. 当前保存的数据格式

每次测量结果会保存为 `CSV` 文件。  
当前保存格式为两列数据：

```text
wavelength,value
```

文件内容不带表头，保存内容是从 `PR788` 读取并解析后的光谱数据。

## 7. 结果区怎么看

### 1. File / Path

显示当前这次测量保存后的文件名和路径。

### 2. sRGB Preview

显示当前测量结果换算后的 `sRGB` 预览颜色。

说明：

- 这是一个视觉参考色块
- 用于快速判断当前测量光色的大致观感
- 颜色由 `colour` 库根据当前测量结果计算得到

### 3. SPD Preview 380-780 nm

显示当前测量的 `SPD` 曲线。

特征：

- 横坐标为 `380–780 nm`
- 白色包络线表示光谱轮廓
- 曲线下方做了按波长的彩色填充

这张图主要用于现场快速查看当前光谱分布形状。

### 4. CIE 1931 x y

显示当前测量的 `xy` 色坐标。

### 5. CIE 1976 u'v'

显示当前测量的 `u'v'` 坐标。

### 6. CIE XYZ

显示当前测量结果计算得到的 `XYZ` 值。

### 7. CCT (Ohno 2013)

显示当前测量结果的相关色温 `CCT`。  
当前算法使用：

```text
Ohno 2013
```

### 8. Luminance

显示当前测量结果的亮度，界面中按 `nit` 显示。

### 9. Tint (Duv, Ohno 2013)

显示当前的偏色程度，当前使用：

```text
Duv
```

通常可以这样理解：

- `Duv > 0`：偏绿
- `Duv < 0`：偏洋红 / 品红
- 越接近 `0`：越接近黑体轨迹

## 8. 当前测量命令说明

当前前端不再暴露测量命令设置。

程序内部固定使用：

```text
M5
```

也就是说每次测量都会向 `PR788` 发送固定的测量命令，不需要用户在界面里手动修改。

## 9. 常见问题

### 1. 程序能打开，但串口列表为空

请检查：

- 设备是否已接入
- 驱动是否正常
- 是否需要点击 `Refresh`

### 2. 连接失败

请检查：

- 当前选择的串口是否正确
- 串口是否被其他程序占用
- `PR788` 是否已经正常上电

### 3. 测量等待很久

这不一定是错误。  
当前程序默认会耐心等待 `PR788` 返回完整数据，特别是在某些读数较慢的场景下，等待时间可能明显变长。

### 4. 颜色预览和实际观感不完全一致

这是正常的。  
`sRGB Preview` 只是基于测量结果做的屏幕参考显示，不等同于真实发光体在实际环境中的肉眼观感。

## 10. 当前推荐使用方式

如果你只是进行日常测量，推荐这样使用：

1. 启动程序
2. 选择串口并连接
3. 设置输出目录
4. 使用 `{counter}_{timestamp}` 作为文件名模板
5. 打开自动编号
6. 连续点击 `Capture Measurement`
7. 在右侧查看 `SPD`、`CCT`、`Tint` 和颜色预览

这样是当前最稳定、最直接的使用路径。
